# AI Log Anomaly Detector

URL http://54.159.106.74:8000/docs

AI(機械学習)を用いてWebリクエスト(ログ)の異常をリアルタイムに検知し、Discordへ通知するバックエンドシステムです。AIを組み込んだこのシステムによって未知のサイバー攻撃にも対応できます。

## 概要
大量のWebアクセスログから「普段とは違う構造パターン」を機械学習モデルで解析・識別します。異常を検知した際は、メイン処理のレスポンス性能を落とさないよう非同期のバックグラウンドタスクとして処理され、Discord Webhookを通じて管理者に即座にアラート送信されます。

## 技術スタック
- **Language:** Pyhon 3.11
- **Framework:** FastAPI 
- **AI/ML:** Scikit-learn (Isolation Forest), Pandas
- **Database:** PostgreSQL, SQLAlchemy (ORM)
- **Infrastructure:** Docker, Docker Compose, AWS EC2
- **Notification:** Discord Webhook API

## テストの実行方法
DATABASE_URL=sqlite:///./test.db python3 -m pytest

## 🛠️ 技術選定理由とトレードオフの評価

本プロジェクトでは、単に「動くものを作る」だけでなく、プロダクション環境での運用、インフラコスト、開発効率を考慮し、競合する他の主要技術との比較を行った上で選定しています。

### 1. 言語・Webフレームワーク
* **選定技術: Python 3.11 + FastAPI**
* **不採用技術: Python + Flask 
    * **比較と選定理由:**
        * 機械学習ライブラリ（Scikit-learn）をシームレスにバックエンドに組み込むため、言語はPython一択としました。
        * **Flaskとの比較:** Flaskは同期処理が基本であり、リクエストごとにスレッドを消費するため、通知処理などのI/O待ちでサーバー全体がボトルネックになります。FastAPIは非同期（`async/await`）を標準サポートしており、非常に軽量です。
    * **実務的メリット:** FastAPIに標準搭載されている `BackgroundTasks` を利用することで、メインのログ受信処理・レスポンスを最優先（即座に `200 OK` を返却）にしつつ、通知処理を裏側で完全に非同期処理する「疎結合な設計」を最小のコード量で実現できました。

### 2. AI / 機械学習アルゴリズム
* **選定技術: Scikit-learn (Isolation Forest)**
* **不採用技術: ロジスティック回帰 などの教師あり学習 
    * **比較と選定理由:**
        * **教師あり学習との比較:** ロジスティック回帰などの分類モデルは、過去のデータの境界線を学習します。そのため、まだ世に出ていない脆弱性を突く「ゼロデイ攻撃」や、未知の変則的なアクセス（振る舞い）が来た場合に対応できず、すり抜けるリスクが非常に高いです。
    * **実務的メリット:** `Isolation Forest` は「正常の骨組み」だけを覚えさせ、そこから外れたものを一括で「孤立」として検知する教師なし学習です。これにより、**未知のサイバー攻撃の振る舞いを網羅・検知できる**システム構成を実現しました。

### 4. データベース
* **選定技術: PostgreSQL + SQLAlchemy (ORM)**
* **不採用技術: SQLite **
    * **比較と選定理由:**
        * **SQLiteとの比較:** SQLiteはファイルベースの軽量なデータベースであり、同時書き込み（Write）のロックが発生するため、大量のアクセスログが秒間で秒間何百件も押し寄せるWebアプリケーションのバッファとしては完全に力不足です。
    * **実務的メリット:** 本番環境を見据え、並行処理性能と堅牢性に優れた `PostgreSQL` を採用。また `SQLAlchemy` の導入により、生のSQLを発行する際のSQLインジェクション脆弱性をコードレベルで排除しました。

### 5. インフラ・デプロイ
* **選定技術: AWS EC2 + Docker / Docker Compose**
* **不採用技術: AWS Lambda (Serverless) **
    * **比較と選定理由:**
        * **AWS Lambdaとの比較:** サーバーレス環境はコスト効率が良い反面、リクエストが発生するたびにコンテナを起動する「コールドスタート」が発生します。特に本システムは `scikit-learn` などの重いライブラリや `.pkl` ファイルをメモリに常駐（ロード）させて推論するため、コールドスタートの遅延が数秒単位で発生し、リアルタイム監視という要件を満たせません。
    * **実務的メリット:** `Docker` でコンテナ化することにより、ローカルで構築した機械学習の実行環境を1ミリも差異なく `AWS EC2` 上へデプロイ可能にしました。常時起動のEC2上でFastAPIプロセスを待機させているため、コンテナの起動オーバーヘッドがなく、ログ到達から推論までをミリ秒の低レイテンシで処理します。
